# Follow-up vs Approve Workflow

## What it is
Two different approaches to handling old booking inquiries. "Approve" immediately creates/confirms a reservation and blocks the dates. "Follow-up" sends a message to the guest asking if they're still interested — without changing anything. The follow-up approach is safer because it avoids creating reservations that guests never intended to complete.

## Tourism Translation
**The difference between holding a restaurant table vs. calling to ask "are you still coming?"**

**Approve approach**: A guest called last week asking about a table for Saturday. You immediately reserve the table, blocking it from other diners. If they never show up, you lost that table for the evening.

**Follow-up approach**: Instead, you call them back: "Hi! Just checking — are you still planning to join us Saturday? These spots fill up fast, so just let us know." If they say yes, THEN you reserve. If they ghost you, no harm done — the table stays open for someone else.

## When to use each
- **Follow-up first**: For old inquiries (more than a few days old), especially from Airbnb where approval doesn't actually confirm the booking
- **Approve directly**: Only when a guest has actively confirmed they want to book AND the dates are available
- **Decline**: When dates are already taken by another confirmed guest

## The Missy Rule
"First come, first serve. Don't block dates for inquiry guests. Send them a message, and if they don't respond, move on."
